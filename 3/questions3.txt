1. Следует придерживаться структуры (проверка предусловий, трассировка/логирование, рабочий код, проверка постусловий).
Если функция получается слишком большой следует разбить ее на несколько функций.
Также не следует использовать перегрузку функции с аргументами по умолчанию.
2. При компилировании для оптимизации работы программы (по памяти, времени или др) компилятор может
 вместо вызова функции скопировать ее код и вставить в код программы в нужном месте.
3.Значение по умолчанию могут иметь только аргументы функции, находящиеся справа,
 то есть справа от них не должно быть аргументов без значения по умолчанию.
4. Выбор перегруженной функции разрешается на основании набора передаваемых аргументов.
5. "Состояния" в функциях можно обеспечить при помощи static объектов, глобальных переменных
 или при передаче аргументов через ссылки/указатели.
"Состояния" в лямбда-выражениях можно обеспечить при захвате переменных по ссылке.
